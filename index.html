Redes de Computadores

Conceito

A espinha dorsal de toda infraestrutura moderna. É a interconexão de dispositivos que permite o compartilhamento de recursos e a troca de informações seguindo modelos arquitetônicos padronizados (como o Modelo OSI e TCP/IP).

Exemplo Prático: Quando você acessa um site, o DNS traduz o nome (ex: google.com) para um IP. O navegador faz uma requisição HTTP/HTTPS. O tráfego desce pelas camadas até a Camada Física, passa por Switches (redes locais) e Roteadores (internet) até o destino.

Conceitos-Chave (Camadas e Equipamentos)

  • Camada Física: Onde a sequência de bits é convertida em sinais eletromagnéticos (cabos, rádio).
  • Camada de Enlace: Onde o Switch opera aprendendo endereços MAC. A PDU (unidade de dados) desta camada é o Quadro (Frame). Aqui também se configuram as VLANs (802.1Q) para segmentação.
  • Camada de Rede: Onde ocorre o Roteamento através de IPs e do Roteador. A PDU é o Pacote. (Protocolos principais: IPv4, IPv6, ICMP).
  • Camada de Transporte: Garante a entrega fim-a-fim entre processos nos sistemas hospedeiros (como o Computador). A PDU é o Segmento.

Protocolos e Fundamentos Adicionais

  • TCP vs UDP: O TCP garante entrega confiável e controle de fluxo, enquanto o UDP é rápido (sem conexão) e muito usado em aplicações de tempo real.
  • Roteamento: Pode ser interno usando algoritmos como o de Dijkstra (OSPF) ou protocolos de borda (BGP) para comunicação entre diferentes Sistemas Autônomos (AS).
  • ICMP: Protocolo de mensagens de controle. É a base de funcionamento de comandos de diagnóstico como o ping e o traceroute.

Arquitetura de Aplicações

Conceito

A forma como os sistemas são estruturados para se comunicar na camada de aplicação define o desempenho e a escalabilidade dos serviços web.

Modelos Principais

  • Client-Server (Cliente-Servidor): Múltiplos terminais (clientes) acessam um ponto centralizado (servidor). É a arquitetura dominante para serviços como a Web (HTTP), transferência de arquivos (FTP) e E-mail.
  • Peer-to-Peer (P2P): Permite a comunicação direta entre os diversos terminais da rede sem a necessidade de um servidor central. Muito utilizado em compartilhamento de arquivos, como o BitTorrent.

Virtualização

Conceito

A virtualização é a tecnologia que permite a criação de representações baseadas em software (virtuais) de recursos físicos. Isso maximiza o uso do hardware, reduz custos operacionais e facilita a recuperação de desastres.

Observação Importante: A virtualização é a base de funcionamento da Computação em Nuvem. Sem a capacidade de fatiar um servidor físico gigantesco em múltiplas instâncias menores, a Nuvem como conhecemos não existiria.

Conceitos-Chave

  • Hypervisor: O "Monitor de Máquina Virtual". O Tipo 1 (Bare-Metal) roda direto no hardware físico (ex: VMware ESXi). O Tipo 2 roda sobre um sistema operacional comum (ex: VirtualBox).
  • Máquinas Virtuais (VMs): Virtualizam o hardware completo e requerem um Sistema Operacional convidado (Guest OS) inteiro para cada instância.
  • Containers (ex: Docker): Mais leves e rápidos que as VMs. Virtualizam a nível de Sistema Operacional, compartilhando o Kernel do hospedeiro e isolando apenas as bibliotecas e a aplicação (usando namespaces e cgroups).
  • Live Migration: Capacidade de mover uma VM em execução de um servidor físico para outro com zero interrupção.

Computação em Nuvem (Cloud Computing)

Conceito

A entrega de recursos de TI sob demanda (computação, armazenamento, banco de dados) via Internet, com precificação baseada no uso (pay-as-you-go). Elimina a necessidade de manter data centers físicos on-premise.

Modelos de Implantação: A nuvem Pública compartilha recursos (AWS, Azure). A Privada é dedicada a uma única organização. A Híbrida interliga a rede local segura com o poder de escala da nuvem pública (geralmente via VPN ou links dedicados).

Modelos de Serviço

Modelo O que é gerenciado pelo Cliente? Exemplo
IaaS (Infraestrutura) Aplicações, Dados, SO, Middleware. (O provedor cuida só do hardware e rede física). Amazon EC2, Azure VMs
PaaS (Plataforma) Apenas Aplicações e Dados. (Ideal para desenvolvedores focarem em código). Google App Engine, Heroku
SaaS (Software) Apenas o uso. (Tudo é gerenciado pelo provedor e entregue pronto no navegador). Microsoft 365, Gmail

Conceitos-Chave

  • Escalabilidade Elástica: Capacidade de aumentar (scale up/out) ou diminuir recursos em tempo real de acordo com a demanda de tráfego.
  • Alta Disponibilidade (HA): Arquitetura projetada para permanecer operacional continuamente, utilizando múltiplas Zonas de Disponibilidade (datacenters separados fisicamente).
  • Orquestração: Ferramentas como o Kubernetes são usadas para gerenciar e escalar milhares de containers de forma automatizada em ambientes de microsserviços.

Segurança e VPN

Conceito

Em ambientes corporativos e de sistemas web, proteger os perímetros da rede e garantir a criptografia dos dados em trânsito são os pilares para uma operação confiável e segura.

Conceitos-Chave

  • Firewall Stateful (Inspeção de Estados): Diferente de filtros simples, ele monitora ativamente as conexões nas camadas de rede (L3) e transporte (L4), entendendo o estado e o fluxo da comunicação para aplicar políticas rigorosas.
  • VPN Site-to-Site: Uma Rede Privada Virtual que estabelece um túnel criptografado pela Internet. É utilizada para conectar filiais geograficamente distantes como se estivessem dentro da mesma rede local (LAN).
  • Protocolo TLS: Sucessor do SSL, é essencial para o HTTPS moderno. Ele garante tanto a confidencialidade (através de criptografia) quanto a integridade dos dados trafegados entre o cliente e o servidor da aplicação.
  • Ataques DoS / DDoS: (Distributed Denial of Service). Ataques cujo objetivo primário não é roubar dados, mas sim sobrecarregar um servidor ou link de rede com tráfego massivo, tornando a infraestrutura indisponível para usuários legítimos.
